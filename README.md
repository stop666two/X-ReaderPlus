# X-ReaderPlus

> 一个完全脱机的现代化多格式阅读器。不做花活，只做实事。

---

## 这是什么

X-ReaderPlus 是一个跑在你电脑本地的桌面阅读器。我写它的初衷很简单：我有一堆 EPUB、TXT、Markdown、PDF、漫画压缩包撒在硬盘各个角落，Windows 自带的阅读器要么格式不全，要么界面丑，要么非要联网登录。Calibre 功能是强，但那个管理逻辑和界面实在不是给"打开—阅读—关掉"这种日常场景设计的。其他几个开源的 Electron 阅读器我也试过，不是年久失修就是缺胳膊少腿。

所以就自己写了一个。所有的数据、书籍、笔记、标注全部存在你自己的硬盘上，不经过任何服务器。你甚至可以拔掉网线，它一样跑得飞快。

核心思路就三条：**格式通吃、隐私至上、读得舒服**。

## 功能

### 书架

- **网格 / 列表双视图切换**——网格模式能看到封面缩略图（EPUB 自动提取），列表模式信息密度更大，想怎么翻怎么翻。
- **拖拽导入**——把文件或文件夹往书架一拖就行，支持批量，几十本书一起扔进去也不会卡。
- **搜索和标签筛选**——书多了可以按书名、作者搜，也能打标签。标签是自定义的，随便你怎么分类，比如"技术""小说""待读""垃圾"（不是）。
- **排序**——最近添加、书名、作者，三种排序，够用了。
- **批量管理**——多选然后批量删除，清架子的时候不用一个一个点。
- **书籍详情编辑**——可以给书打分（1-5 星）、写短评、加标签，算是一个轻量级的读书笔记入口。
- **删除进回收站**——不小心删了没关系，先进回收站，确认不要了再彻底清掉。

### 阅读器

这才是重头戏。阅读器是围绕"沉浸感"设计的：

- **分章节阅读**——EPUB 自带目录，TXT 自动按"第X章/第X节/Chapter"等模式切分，Markdown 按 `#` 标题切。一本书拆成章，一章一章读，不会被几百页正文淹没。
- **章节导航**——上一章/下一章，四种方式随便用：工具栏按钮、键盘左右方向键、点屏幕左右边缘、底部章节滑块。
- **排版微调**——字号（12-32px）、行距（1.2-3.0）、段距（0-40px）、缩进（0-4em）、字重（300-700）、字体、边距全部可调。每项都有滑块，所见即所得。
- **3 种阅读主题**——明亮（白底黑字）、暗黑（深灰底浅灰字）、护眼（米黄底褐字）。我个人最喜欢护眼模式，晚上读两个小时眼睛不酸。
- **Ctrl+滚轮缩放字号**——跟浏览器一样，肌肉记忆不用改。缩放范围 -50% 到 +100%。
- **工具栏自动隐藏**——鼠标不动 4 秒后工具栏自动收起（可调），给你一整屏纯粹的阅读空间。鼠标稍微动一下工具栏就出来，不用按键。
- **全文搜索**——按 Ctrl+F 打开搜索栏，输入关键词，所有匹配项高亮显示，可以逐个跳转。搜索结果实时更新。
- **目录面板**——侧边栏弹出当前书籍的完整目录，点击直接跳转。
- **书签面板**——随时添加/删除书签，记录重要的阅读位置。
- **选中文字操作**——划词后弹出菜单：
  - **高亮标注**：6 种颜色（黄/绿/蓝/红/橙/紫），像用荧光笔一样，可以在一章里画多种颜色。
  - **笔记**：给选中的文字写批注，就是读书时在空白处写字那种感觉，不过是数字版的。
  - **复制**：常规操作。
  - **查词典**：划词直接查在线英文词典（使用 dictionaryapi.dev，需你手动确认一次）。
- **标注管理**——查看当前章节所有标注，可以编辑笔记内容、删除标注。
- **自动滚屏**——点一下自动播放按钮，页面以可调速度自动往下滚，解放双手。适合躺着读或者吃东西的时候用。
- **滚动位置记忆**——每章独立记住你滚到哪里，换章再回来还是那个位置。关闭书籍再打开，回到上次退出的地方。
- **阅读进度条**——底部显示当前章节的阅读进度，一目了然。
- **虚拟滚动**——长章节（比如一整本 TXT 就一章）不会卡，只渲染你屏幕附近的内容，上万行的章节也流畅。

### 笔记

独立于阅读器的全局笔记视图：

- 所有标注集中展示，按书归类。
- 按书筛选、按关键词搜索。
- 直接编辑笔记内容，不用回到阅读器。
- 点击跳转到对应书籍，自动打开到标注所在章节。
- 删除标注也是在这里统一管理。

### 词典

- **书库全文搜索**——跨书搜索，搜文章内容、章节标题、文件名、标签，全维度覆盖。
- **在线英文词典**——调用 dictionaryapi.dev，选中单词后可以查释义、音标、例句。首次使用需要你手动确认一次联网请求，之后就不会再问了。
- **最近查询记录**——查过的词会保留历史，方便回头复习。

### 统计

- 总藏书量、总字数、格式分布饼图。
- 看着自己的阅读数据增长还是挺有成就感的。

### 历史记录

- "最近阅读"列表，快速回到上次在读的书。
- 按最后阅读时间排序。

### 回收站

- 被删除的书进回收站而非直接消失。
- 可以随时恢复。
- 清空回收站就是真删了，删之前会二次确认。

### 设置

除了阅读排版参数外：

- **主题切换**——明亮/暗黑/护眼，随时能换。也有全局快捷键 Ctrl+T 一键切换。
- **数据管理**：
  - **导出备份**：把所有数据（书架元数据 + 章节内容 + 标注 + 书签 + 设置）打包成一个加密 JSON 文件，可以存到任意位置。
  - **导入恢复**：从备份文件恢复全部数据。换了电脑也不怕。
  - **刷新元数据**：重新扫描书架上的文件，更新文件大小等信息。
  - **清除全部数据**：初始化到出厂状态，重置前需要二次确认。
- **安全设置**——PIN 码保护（详见下方"安全"一节）。

### 桌面端特性

因为是 Electron 套壳，所以能做一些浏览器做不了的事：

- **无边框窗口**——没有 Windows 自带那种粗边框和难看的标题栏，整个窗口就是你自己的。圆角柔和，看起来像一个正经的现代应用。
- **自定义标题栏**——标题栏集成主题切换、最小化、最大化、关闭按钮，与内容区融为一体。
- **老板键（Ctrl+B）**——一键隐藏窗口，再按一次恢复。关键时刻救命。
- **命令面板（Ctrl+K）**——全局搜索框，可以搜书、搜功能入口，类似 VS Code 的 Ctrl+Shift+P。
- **全局快捷键**——Ctrl+T 切换主题，不用切回应用就能操作。

## 安装

### Windows

去 [Releases](https://github.com/stop666two/X-ReaderPlus/releases) 下载最新版，根据你的系统选架构：

| 平台 | 安装版 | 便携版 |
|------|--------|--------|
| **Windows x64** | `X-ReaderPlus.Setup.0.2.0.exe` | `X-ReaderPlus-0.2.0-win.zip` |
| **Windows ia32** | — | `X-ReaderPlus-0.2.0-ia32-win.zip` |
| **Windows arm64** | — | `X-ReaderPlus-0.2.0-arm64-win.zip` |

- **安装版**：双击安装，可选安装位置，开始菜单创建快捷方式
- **便携版**：zip 解压到任意文件夹，双击 `X-ReaderPlus.exe` 即用，数据存在解压目录，可放 U 盘

如果 SmartScreen 弹窗，点"更多信息"→"仍要运行"即可。

### macOS

去 [Releases](https://github.com/stop666two/X-ReaderPlus/releases) 下载 `X-ReaderPlus-0.2.0-arm64.dmg`，双击打开，把 X-ReaderPlus 拖进 Applications 文件夹。

首次打开时系统会提示"无法验证开发者"，去 **系统设置 → 隐私与安全性** 里点"仍要打开"即可（同理，没买 Apple 开发者证书）。

支持 macOS 11 (Big Sur) 及以上。

### Linux

去 [Releases](https://github.com/stop666two/X-ReaderPlus/releases) 下载 `X-ReaderPlus-x.x.x.AppImage`，给执行权限直接跑：

```bash
chmod +x X-ReaderPlus-*.AppImage
./X-ReaderPlus-*.AppImage
```

主流发行版（Ubuntu 20.04+、Debian 11+、Fedora 36+、Arch）都能跑。想集成到系统菜单可以用 AppImageLauncher。

### 系统要求

- Windows 10+ / macOS 11+ / Linux
- x64 架构（Mac M1/M2/M3 通过 Rosetta 2 转译）
- 不需要任何运行时依赖，不需要联网
- 内存：空载约 150MB，具体看加载了多少书
- 硬盘：应用约 200MB，书籍数据另外算

## 使用指南

### 导入书籍

方法一：打开 X-ReaderPlus，直接把文件或文件夹拖到书架区域。  
方法二：点击书架上的 "+" 按钮，弹出系统文件选择框，多选导入。  
方法三：如果你是命令行党，可以把书放到任意位置，回头用"打开文件"方式直接读。

支持一次导入几十本书，每本都会解析出元数据（书名、作者、封面）并存入数据库。大文件（>500MB）会提示你，EPUB 内部超过 10000 个文件的也会拒绝——这是故意的限制，防止有人拿整站打包的 EPUB 把程序搞崩。

### 开始阅读

书架上点任意一本书的封面或书名，就进入阅读器。首次打开会自动跳到第一章或记忆的上次位置。

阅读器内：
- 鼠标不动 4 秒，工具栏自动收起，给你全屏沉浸空间。
- 移动鼠标或按键盘，工具栏重新出现。
- 点屏幕左边缘或按左方向键 = 上一章；右边缘或右方向键 = 下一章。
- Ctrl+F 当前章节搜索，输入关键词回车。
- 选中文字弹出操作菜单。

### 划词标注

选中文字后出现菜单：
1. 选一种高亮颜色，文字就被标记了。
2. 点击"笔记"，写上你的批注，支持多行。
3. 标注会自动保存，在"笔记"页面可以看到全局列表。

### 笔记管理

左侧导航栏点"笔记"，进入全局标注视图：
- 可以按书筛选，也可以关键词搜索。
- 点击任一标注，跳转到对应书籍该章节的精确位置。
- 笔记内容可以在这里直接编辑，不用回到阅读器。

### 书签

阅读器工具栏点书签按钮，在弹出面板里点"添加书签"，当前位置就被记录下来了。书签会显示所在章节和段落偏移，点一下就能跳回去。

### 数据备份

**强烈建议定期备份。** 在"设置"→"数据管理"里点"导出备份"，选个安全的地方（推荐存到云盘或外置硬盘）。备份文件是 AES-256-GCM 加密的 JSON，就算别人拿到也没有你的 PIN 码解不开。

恢复时同样在设置里点"导入恢复"即可。

## 安全

X-ReaderPlus 对安全的态度是认真的，因为我把自己的书也放在里面。

### 数据完全脱机

所有数据存储在本地 SQLite 数据库中，不经由任何网络传输。唯一的联网请求是在线词典查询（dictionaryapi.dev），而且首次使用需要用户手动确认，之后可以随时在设置里关闭。除此之外，程序没有任何埋点、遥测、数据收集。

### PIN 码保护

你可以在设置里启用 PIN 码锁。启用后：

1. **SHA-256 哈希 + PBKDF2 密钥派生**——你的 PIN 码经过 60 万次 PBKDF2 迭代后才用于验证和加密，暴力破解成本极高。
2. **AES-256-GCM 加密**——所有章节内容、书签、标注都用你的 PIN 码派生的 AES-256-GCM 密钥加密存储在 SQLite 中。不知道 PIN 码 = 无法解密 = 数据不可读。
3. **5 次错误锁定 30 秒**——防止暴力尝试。
4. **密钥不落盘**——加密密钥只在内存中保留，程序关闭就消失。

简单理解：你设的 PIN 码就是唯一钥匙，忘了就真的打不开了，我也没办法帮你找回。所以密码别太复杂到自己都记不住，但也别用"1234"这种。

### HTML 内容净化

因为你导入的书（尤其是 EPUB 和 HTML）可能包含未知来源的脚本，所有渲染到页面的 HTML 内容都会经过 DOMPurify 净化器过滤。EPUB 解析阶段也会主动移除 `<script>`、`<style>`、`<svg>` 等可能带来 XSS 风险的元素。生产环境使用 `app://` 自定义协议加严格 CSP 策略，double 保险。

### CSP 策略（生产环境）

```
default-src 'self'
script-src 'self' 'unsafe-inline'
style-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net
font-src 'self' https://cdn.jsdelivr.net
img-src 'self' data: blob:
media-src 'self'
connect-src 'self' https://api.dictionaryapi.dev
worker-src 'self' blob:
child-src blob:
base-uri 'self'
form-action 'none'
frame-ancestors 'none'
object-src 'none'
```

另外 Electron 主进程配置了 `sandbox: true` + `contextIsolation: true` + `nodeIntegration: false`，并注册了 `will-navigate` 事件阻止外部 URL 导航。翻译成人话：除了你自己电脑上的资源和词典 API，其他地方一概不准通信。

## 构建

如果你想从源码自己编译：

### 环境准备

```bash
# 需要 Node.js 18+ 和 npm
node -v   # 确认版本 >= 18
npm -v    # 确认版本 >= 9

# 克隆仓库
git clone https://github.com/stop666two/X-ReaderPlus.git
cd x-reader-plus

# 安装依赖
npm install
```

### 开发模式

```bash
npm run dev
```

这会启动 Vite 开发服务器（端口 5173），同时启动 Electron 窗口。修改代码后热更新即时生效。不过 Electron 主进程的修改需要手动重启。

### 生产构建

```bash
npm run build
```

这个命令做了三件事：
1. `vue-tsc --noEmit`——TypeScript 类型检查，确保没有类型错误。
2. `vite build`——Vue 前端打包到 `dist/`。
3. `electron-builder`——打包成对应平台的可执行安装包，输出到 `release/`。

构建产物：
- Windows 安装版：`release/X-ReaderPlus.Setup.x.x.x.exe`
- Windows 便携版：`release/X-ReaderPlus-x.x.x-win.zip`
- macOS：`release/X-ReaderPlus-x.x.x-arm64.dmg`
- Linux：`release/X-ReaderPlus-x.x.x.AppImage`

### 仅构建前端

```bash
npm run build:vite
```

如果你只想改前端 UI 部分，用这个就够了，省去 electron-builder 的时间。

### 技术栈

| 层面 | 技术 |
|------|------|
| 前端框架 | Vue 3 (Composition API) |
| 状态管理 | Pinia |
| UI 框架 | Vuetify 3 (按需引入) |
| 类型系统 | TypeScript (strict 模式) |
| 构建工具 | Vite 6 |
| 桌面壳 | Electron 42 |
| 本地数据库 | **SQLite (better-sqlite3)** — WAL 模式，IPC 通信 |
| EPUB 解析 | epub.js (Worker 线程) |
| PDF 解析 | pdfjs-dist |
| Office 文档 | Mammoth.js (DOCX) |
| Markdown | marked |
| 加密 | Web Crypto API (AES-256-GCM + PBKDF2) |
| HTML 净化 | DOMPurify |
| 国际化 | vue-i18n (中文/英文，11 个视图迁移中) |
| 压缩文件 | JSZip (ZIP/CBZ 处理) |
| 日期处理 | dayjs |
| 图标 | @mdi/font (Material Design Icons) |

## FAQ

### Q: 为什么要再写一个阅读器？市面上不是已经有很多了吗？

确实，市面上阅读器不少。但我没找到一个同时满足以下条件的：完全离线、支持 EPUB+MOBI+PDF+DOCX+漫画、界面现代、有划词标注和笔记功能、支持 PIN 码加密、跨平台。每个单独要求可能都有某款软件能做到，但全部凑齐的，我没找到。所以就自己写了。另一个原因是，我不信任那些需要登录、会上传你阅读数据的"免费"阅读器。我的书架是我的隐私。

### Q: 能看 PDF 吗？效果怎么样？

能，基于 pdfjs-dist 做文本提取和渲染。不过实话实说，PDF 不是 X-ReaderPlus 的主战场。如果你是重度 PDF 用户（教材、论文、扫描件），建议还是用专门的 PDF 阅读器。X-ReaderPlus 的 PDF 阅读适合"小说类 PDF"或"文字为主、排版简单的 PDF"。复杂的多栏排版、学术论文格式可能显示不够完美。

### Q: 能同步到手机吗？

当前版本不能。X-ReaderPlus 是桌面端应用，没有移动版本。计划中会加入 WebDAV 同步功能，这样你可以在不同电脑之间同步阅读进度和标注。至于手机上，你可以用备份/恢复功能手动迁移数据，但体验确实不是为移动端设计的。

### Q: 支持 Kindle DRM 的书吗？

不支持，也不打算支持。DRM 破解涉及法律灰色地带，X-ReaderPlus 不触碰这部分。如果你有无 DRM 的 MOBI/AZW3 文件（比如个人文档、公开版权书籍），那可以正常阅读。

### Q: 我的书太多了，书架会卡吗？

书架用了虚拟滚动，即使几百本书也能流畅滚动。每本书的元数据（书名、作者、封面缩略图）都做了索引。实际测试过 500+ 本书，打开书架一秒内显示，滚动无延迟。

### Q: 能自定义字体吗？

字体可以从系统已安装字体中选择。在设置→阅读设置→字体下拉框里能看到你系统里所有字体。如果你装了"霞鹜文楷""思源宋体"这类阅读友好的字体，体验会更好。

### Q: PIN 码忘了怎么办？

凉拌。PIN 码是加密的唯一钥匙，设计上就没留后门。忘记了就意味着数据解不开了。建议把 PIN 码记在安全的地方，或者导出备份后，重置应用再导入。不过这也意味着你需要在导出备份时先解密——总之，别忘。

### Q: 项目开源吗？能商用吗？

完全开源，MIT 协议。你可以拿去自己改、自己用、给朋友用。但**禁止直接转卖**——这是底线。我花了大量个人时间把这事做出来免费给大家用，不是让别人拿去套个壳就上架商店卖钱的。二次开发、学习参考、企业内部分发都没问题，就是别干倒卖这种恶心事。

### Q: 有 Bug 怎么办？怎么反馈？

去 GitHub Issues 提，描述清楚你遇到的现象、复现步骤、什么格式的书、文件大概多大。如果方便的话附上截图。我会尽量修，但因为是个人项目，不保证响应速度。

### Q: 会加 AI 功能吗？

不会。AI 要联网，要调 API，要有服务器——这跟 X-ReaderPlus"完全离线"的核心理念冲突。阅读是件安静的事，我不想在上面加一堆花里胡哨的东西。如果你需要 AI 辅助阅读，市面上有其他产品做这个。
